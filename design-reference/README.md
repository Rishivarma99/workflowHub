# Design Reference

Drop the HTML/React screens generated by Claude's design feature here. They are used **only as a
visual + structural reference** for building the real Angular 20 + Tailwind + PrimeNG screens —
this code is **not** shipped or imported into the app.

## How to add a screen

1. In the Claude.ai design artifact, copy/export the generated code (or download the files).
2. Save each screen as its own file or folder here, e.g.:
   ```
   design-reference/
     login.html              (or login.tsx / login.jsx)
     workflow-list.tsx
     workflow-detail.tsx
     install-modal.tsx
     assets/                 (any images/icons the screens use)
   ```
3. Name files after the screen so the mapping to a feature is obvious.

## How it gets used

For each reference screen, the Angular implementation will live under
`frontend/src/app/features/<area>/...` and reuse:
- **Tailwind** for layout/spacing/typography (port the utility classes over),
- **PrimeNG** for interactive components (buttons, inputs, dialogs, tables) where the design uses them,
- the `core/shared/features` structure and naming rules in `../ai-rules/`.

The reference's React/HTML structure and styling are matched; the React-specific bits
(state hooks, JSX handlers) are re-expressed in Angular idioms.

# Design Reference

The HTML/React screens generated by Claude's design feature live in a **sibling folder outside this
repo**, named `workflowHub-design-reference/` (kept next to `workflowHub/` in the same parent
folder). They are used **only as a visual + structural reference** for building the real
Angular 20 + Tailwind + PrimeNG screens — that code is **not** shipped or imported into the app.

```
Projects/
├── workflowHub/                     <- THIS repo
│   └── design-reference/            <- you are here (just this pointer doc)
└── workflowHub-design-reference/    <- the actual reference screens (sibling, outside the repo)
    ├── Workflow Hub.html
    ├── app.jsx / app.css
    ├── screens1.jsx / screens2.jsx
    ├── ui.jsx
    └── data.jsx
```

> If the reference isn't found at `../workflowHub-design-reference/`, ask for it to be placed there
> as a sibling of `workflowHub/`.

## How to add a screen

1. In the Claude.ai design artifact, copy/export the generated code (or download the files).
2. Save each screen into the sibling `workflowHub-design-reference/` folder, named after the screen
   so the mapping to a feature is obvious (e.g. `login.jsx`, `workflow-list.jsx`,
   `workflow-detail.jsx`, `install-modal.jsx`), plus any `assets/` the screens use.

## How it gets used

For each reference screen, the Angular implementation will live under
`frontend/src/app/features/<area>/...` and reuse:
- **Tailwind** for layout/spacing/typography (port the utility classes over),
- **PrimeNG** for interactive components (buttons, inputs, dialogs, tables) where the design uses them,
- the `core/shared/features` structure and naming rules in `../ai-rules/`.

The reference's React/HTML structure and styling are matched; the React-specific bits
(state hooks, JSX handlers) are re-expressed in Angular idioms.

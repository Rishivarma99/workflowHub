# Design Reference

The HTML/React screens generated by Claude's design feature live in a **sibling folder outside this
repo**, named `workflowHub-design-reference/` (kept next to `workflowHub-frontend/` in the same
parent folder). They are used **only as a visual + structural reference** for building the real
Angular 20 + Tailwind + PrimeNG screens — that code is **not** shipped or imported into the app.

```
Projects/
├── workflowHub-frontend/            <- THIS repo
│   └── design-reference/            <- you are here (pointer doc only)
├── workflowHub-backend/             <- API (sibling)
└── workflowHub-design-reference/    <- the actual reference screens (sibling)
    ├── Workflow Hub.html
    ├── app.jsx / app.css
    └── ...
```

> If the reference isn't found at `../workflowHub-design-reference/`, place it as a sibling of this
> repo.

## How it gets used

Angular implementations live under `frontend/src/app/features/<area>/...` using Tailwind, PrimeNG,
and rules from `../ai-rules/`. Match structure and styling from the reference; re-express React/JSX
in Angular idioms.
